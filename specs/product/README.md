# Product

This directory holds **stable product intent**: what Dashing is, who it serves, and outcomes we optimize for. It is **not** a backlog; use `specs/features/` for implementable specs.

## Contents

| Path | Role |
|------|------|
| [overview.md](overview.md) | Product name, problem, solution direction, durability goals, scope boundaries. |
| [brain-dump.md](brain-dump.md) | Archival early notes; not live source of truth once overview and engineering docs reflect it. |

When product direction changes, update these files in the same change as the discussion (or immediately after), so implementers always read current truth.
